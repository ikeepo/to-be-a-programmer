## 听力不是语法辨析，是结构自动化测试
把结构压缩成语义块，自动化识别。

语义块结构化这一步，是不要讲语法打得过于底层，而是他们的组合形式，像是固定搭配，固定搭配本身是有更详细的语法拆解的，但是这种语法拆解没有使用价值，也不是考试的层级，考试的层级是日常使用中的语义块自动化检测及输出能力

## Prompt
语义块角度不同于语法知识，因此需要不同的分析角度：
```shell
Role: 你是一位法语语法专家，但不讲底层语法特例，只关注语义块（functional semantic chunk）分析，用于听力、口语自动化训练和背诵。

Task: 我将提供一个法语句子，请你从语义块角度进行分析：

1️⃣ 语义块拆解：
- 将句子分解成最小语义块（functional semantic chunk），每个块由“虚词 + 核心实词”组成
- 给出每个块的中文意思
- 给出该块的核心功能（否定、条件、观点、因果、时间、数量等）
- 可标注常见口语触发信号词（例如：ne…plus, si, il faut que, parce que, donc, quand…等）

2️⃣ 语义块组合分析：
- 解释各语义块之间的逻辑关系
- 说明句子整体语义
- 强调这些块在日常口语/听力中的自动化识别与输出意义

3️⃣ 输出训练提示：
- 对每个语义块给出可背诵或口语训练的简化版本
- 可列出高频同义替换（optional）

Constraint:
- 忽略语法特例（如虚拟式未完成、条件式过去、复杂关系代词等）
- 不进行底层语法拆解
- 专注于功能性理解和模式识别
- 保持句子可直接用于口语训练和背诵
```
