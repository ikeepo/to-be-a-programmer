# “说”对口语的帮助
口语能力的很大权重是建立在“听”上的，“说”这个动作对“口语能力”的帮助主要在“流利度”，也就是口腔肌肉的灵活运用，除此之外的口语能力，实际是通过“听”这个动作来构建的。

“说”有三层核心功能：

- 激活记忆（最重要）

- 建立“语音→动作”映射

- 自动化

“说”这个动作是把“听”构建的语言能力，自由、灵活、流利调用出来。听决定你“能不能理解语言”，说决定你“能不能使用语言”

口语 = 听（建模） + 说（激活 + 自动化）
# 编程建模方式类比
```shell
听 =
 你看了很多优秀代码（理解了）

说 =
 你自己开始写代码

过程：

第一次写：很慢（激活）

多写几次：越来越顺（强化）

熟练后：不用想直接写（自动化）
```

这让我回想到我之前学习编程，我以为我持续的写python代码就能提升水平，实际上没有；因为没有足够的输入，只输出是无效的.

输出不会创造新结构，只会重复已有结构。

输入决定上限，输出决定转化率。

# 法语修饰成分似乎倾向于后置
`法语是“核心优先 + 修饰位置灵活”`

信息线性展开策略（information flow）,避免让听者等待核心信息.

法语倾向于先交代“核心语义”，再逐步叠加修饰信息，而不是一开始就给完整结构
